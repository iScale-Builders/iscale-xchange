# Governance

iScaleBuilders is a maintainer-led open-source project.

## Maintainers

The project is maintained by iScaleLabs. Maintainers decide what is merged,
released, documented, and deployed.

## Decision Making

Maintainers may accept, request changes to, or reject contributions based on:

- product direction;
- user experience quality;
- maintainability;
- security and privacy risk;
- license and attribution obligations;
- compatibility with the public-repo safety rules;
- operational cost.

Open source does not mean every proposed change will be merged. Maintainers are
not obligated to accept changes that are off-direction, too broad, unsafe, or
not worth the long-term maintenance burden.

## Pull Requests

Small fixes may be reviewed directly as pull requests.

Large changes should begin as issues or discussions before implementation,
especially changes to public UI, schema, authentication, deployment, licensing,
branding, or contribution policy.

Maintainers may squash, edit, or reword commits before merge.

## Releases And Deployments

The public repository is the public source lane. Production deployment is owned
by iScaleLabs and may use private Vercel, database, authentication, email, and
analytics accounts.

Publishing source code does not grant contributors deployment access.

## Security And Conduct

Security reports are handled through `SECURITY.md`.

Community conduct is governed by `CODE_OF_CONDUCT.md`.
